# Quick Start - 3 Steps

## Step 1: Get Your Keys (2 minutes)

### GitHub Token
```
1. Visit: https://github.com/settings/tokens
2. Click "Generate new token (classic)"
3. Select scope: public_repo
4. Copy the token
```

### OpenAI Key
```
1. Visit: https://platform.openai.com/account/api-keys
2. Click "Create new secret key"
3. Copy the key
```

## Step 2: Add to GitHub (1 minute)

Go to: `github.com/GABIs-Hub/Daily-log/settings/secrets/actions`

Click "New repository secret" and add:
- Name: `OPENAI_API_KEY`
- Value: (paste your OpenAI key)

Then add another secret:
- Name: `GABI_GITHUB_TOKEN`
- Value: (paste your GitHub token)

## Step 3: Done! (Let it run)

The workflow runs every day at **9 AM UTC**.

To test it now:
1. Go to Actions tab
2. Click "Daily Developer Log"
3. Click "Run workflow"
4. Wait 1-2 minutes
5. Your README updates automatically!

## Customization (Optional)

### Change the Daily Time

Edit: `.github/workflows/daily-summary.yml`

```yaml
cron: '0 9 * * *'  # Change 9 to different hour
```

Common times:
- `'0 9 * * *'` = 9 AM UTC
- `'0 14 * * *'` = 2 PM UTC  
- `'0 22 * * *'` = 10 PM UTC

### Change the Lookback Period

Edit: `scripts/generate.js`

```javascript
const activities = await fetchUserActivity(GITHUB_USERNAME, 7);
// Change 7 to: 14 (2 weeks), 30 (1 month), etc.
```

### Use Better AI (Optional)

Edit: `scripts/summarize.js`

```javascript
model: openai('gpt-4o-mini'), // Fast, cheap (default)
// Change to:
// model: openai('gpt-4o'), // Better quality
```

## Troubleshooting

**"No activities"?**
- Make sure you have public GitHub pushes
- Check your username: `GITHUB_USERNAME` in scripts/generate.js

**"OpenAI error"?**
- Workflow still works! It shows basic stats without AI
- Add your OpenAI key to GitHub secrets to enable summaries

**Workflow not running?**
- Check Actions tab for error logs
- Try manually: Actions → Daily Developer Log → Run workflow

**README not updating?**
- Check Actions tab for errors
- Verify secrets are properly set
